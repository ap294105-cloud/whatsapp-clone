Client-Side Cryptography (E2EE)

Private keys are stored encrypted on your device using passcode-derived XOR encoding. Text messages and media attachments are fully encrypted before transmitting, ensuring zero-knowledge message delivery.

35 Layers of Hardening

Fortified against exploits with custom Helmet Content Security Policies (CSP), CORS locks, parameter pollution checks, query/socket rate-limiters, DOM sanitizers, frame-busting clickjacking protection, and XSS filtering.

Native Android WebView App

A native Gradle-compiled Android application wrapper. Incorporates network configurations supporting dynamic local server connections and leverages offline-cached files to run in hybrid app environments.

Resilient Offline Queueing

If network connection drops, messages and document files are queued in a local outbox. An animated offline banner notifies user status, flushing outbox queues instantly when socket reconnects.

Advanced Status Stories

Publish textual status updates with preselected background gradients, or share image statuses complete with captions. Features automatic 24-hour story expiration and reply handlers.

Privacy Control Center

Manage visibility settings globally for Last Seen timestamps, Online presence indicators, Profile photos, and bios with Everyone / My Contacts / Nobody filters, plus read receipts.

Demo Server Online

WebSocket server is running. Web clients and Android APK instances will automatically bind to database services.
